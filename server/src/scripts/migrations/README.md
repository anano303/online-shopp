# Migration Scripts

ეს ფოლდერი შეიცავს migration script-ებს, რომლებიც გამოიყენებოდა MongoDB-ში მონაცემების გასასწორებლად.

## მიზანი

ვარიანტების პრობლემების მოგვარება - პროდუქტები რომლებსაც ჰქონდათ არასწორი/ცარიელი ვარიანტები.

## გაშვების თარიღი: 2025-09-10

### Scripts და მათი დანიშნულება:

1. **fix-broken-variants.js** - პირველი migration, რომელმაც იპოვა პროდუქტები ცარიელი attribute array-ებით
2. **clean-empty-variants.js** - აწმენდა ცარიელი string ველების მქონე ვარიანტებს
3. **clean-variant-fields.js** - პირდაპირ აშორებდა არავალიდურ ველებს ვარიანტებიდან
4. **fix-specific-product.js** - კონკრეტული პროდუქტის (ani beroshvilii) გასწორება
5. **remove-incomplete-variants.js** - არაკომპლექტური ვარიანტების მოშორება
6. **remove-empty-variants.js** - ცარიელი ვარიანტების მოშორება
7. **sync-count-in-stock.js** - ვარიანტებისა და countInStock-ის სინქრონიზაცია
8. **inspect-product.js** - debugging script-ი კონკრეტული პროდუქტის შესამოწმებლად

## მიღწეული შედეგები:

- ✅ აღმოიფხვრა "Variant not found" შეცდომები
- ✅ დასინქრონდა countInStock ვარიანტების stock-თან
- ✅ გაიწმინდა ბაზა ცარიელი/არავალიდური ვარიანტებისგან
- ✅ გაუმჯობესდა ვარიანტების მეტჩინგ ლოგიკა კოდში

## გაფრთხილება:

ეს script-ები უკვე გაშვებულია და დასრულებულია. ხელახლა არ გაუშვათ production ბაზაზე!
